Deployment Manual

Complete step-by-step guide to deploying Simplified Dental Clinic Management Systems — from Google account setup through live clinic use. Covers cloud deployment via GitHub Pages, local/offline server mode, backup tools, user roles, and plan licensing.

System Overview

Simplified Dental Clinic Management Systems is a single-file web application that works with two backend modes:

ModeBackendBest For
☁️ CloudGoogle Apps Script + Google SheetsMultiple devices, anywhere access, no PC required 24/7
🖥️ LocalNode.js server on your PC or USBOffline clinics, no internet dependency, fast performance

The app can switch between modes at any time from the Settings → Sync page without losing data.

CYRABELL_DentalMS/ ├── cyrabelldental_v2.html ← Main application (open this in browser) ├── backup-tool.html ← Backup & restore utility ├── cyrabell-presentation.html ← Sales/marketing presentation ├── local-server/ │ ├── server.js ← Node.js local backend │ └── package.json ├── APPS_SCRIPT_GEMINI_OCR.js ← Paste into Google Apps Script └── APPS_SCRIPT_DRIVE_UPLOAD.js ← Paste into Google Apps Script

Google Setup

Step 1.1

Create the Google Sheet

This sheet is the database for the cloud deployment.

  1. Go to sheets.google.com and click Blank spreadsheet.
  2. Rename it: click "Untitled spreadsheet" at the top → type Simplified Dental Clinic Management Systems.
  3. Create the following tabs (click the + at the bottom for each):
    • patients
    • appointments
    • payments
    • notifications
    • reminders
    • reminderLog
    • services
    • serverLogs
  4. Copy the URL from your browser — you will need the Spreadsheet ID (the long string between /d/ and /edit).
💡

The tab names are case-sensitive. Make sure they match exactly as listed above.

Step 1.2

Deploy the Apps Script Backend

  1. In your Google Sheet, click Extensions → Apps Script. A new tab opens.
  2. Delete all default code in the editor.
  3. Open the file APPS_SCRIPT_GEMINI_OCR.js from the Cyrabell folder and copy all its contents.
  4. Paste it into the Apps Script editor.
  5. Find the line near the top: 
    const SPREADSHEET_ID = 'YOUR_SPREADSHEET_ID_HERE';
    Replace YOUR_SPREADSHEET_ID_HERE with the ID you copied in Step 1.1.
  6. Click the 💾 Save icon (or Ctrl+S).
  7. Click Deploy → New deployment.
  8. Click the gear ⚙️ next to "Select type" → choose Web app.
  9. Fill in:
    • Description: Simplified Dental Clinic Management Systems v1
    • Execute as: Me
    • Who has access: Anyone
  10. Click Deploy. Google will ask you to authorize — click Authorize access and follow the prompts.
  11. Copy the Web app URL that appears — it looks like:
    https://script.google.com/macros/s/AKfycb.../exec
    Save this URL — you will need it in Phase 2.
⚠️

Every time you edit the Apps Script code you must create a New Deployment (not "Manage deployments") to get an updated URL. The old URL will stop working.

Step 1.3

Add the Drive Upload Script (for photo/file uploads)

  1. In Apps Script, click + next to "Files" on the left panel → Script.
  2. Name the new file DriveUpload.
  3. Open APPS_SCRIPT_DRIVE_UPLOAD.js from the Cyrabell folder, copy all its contents, and paste into the new script file.
  4. Save. The function is automatically available to the main script — no re-deployment needed for this step.
Step 1.4

Get Your Gemini API Key Required for AI + OCR

  1. Go to aistudio.google.com/apikey.
  2. Click Create API key → select your Google project → Create API key in existing project.
  3. Copy the API key (starts with AIza...).
  4. In your app, go to Settings → Sync → scroll to the Gemini API Key card.
  5. Paste the key and click ☁️ Save to Server. The key is stored securely in Google's PropertiesService — it is never visible in the browser after saving.
  6. Click 🗑 Clear Local Copy to remove it from your device.
🔒

The Gemini key is stored server-side only. The app only receives a true/false confirmation that a key exists — the actual key is never sent back to the browser.

App Deployment (GitHub Pages)

Step 2.1

Upload the App to GitHub

  1. Log in to github.com.
  2. Click New repository. Name it e.g. cyrabell-dental. Set it to Public. Click Create repository.
  3. On your computer, open a terminal in the Cyrabell folder and run: 
    git init
git add cyrabelldental_v2.html index.html
git commit -m "Initial deploy"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/cyrabell-dental.git
git push -u origin main
  4. Replace YOUR_USERNAME with your GitHub username.
💡

Only push cyrabelldental_v2.html and index.html. Do NOT push local-server/ or any file containing API keys.

Step 2.2

Enable GitHub Pages

  1. In your GitHub repository, click Settings (top menu).
  2. Click Pages in the left sidebar.
  3. Under Source, select Deploy from a branch.
  4. Set branch to main, folder to / (root). Click Save.
  5. Wait 1–2 minutes. Refresh the page. Your site URL will appear:
    https://YOUR_USERNAME.github.io/cyrabell-dental/cyrabelldental_v2.html
  6. Open this URL in your browser to confirm the app loads.
✅

GitHub Pages is free and serves the app over HTTPS. Bookmark this URL — share it with all clinic staff for access on any device.

Step 2.3

Bake the GAS URL Into the App (Zero-Config for Staff)

This optional step means staff never need to type the GAS URL — it's pre-filled automatically.

  1. Open cyrabelldental_v2.html in a text editor.
  2. Search for APP_DEFAULTS (around line 3388).
  3. Find this line: 
    GAS_URL: '',  // ← paste your Apps Script /exec URL here
  4. Paste your Apps Script URL between the quotes: 
    GAS_URL: 'https://script.google.com/macros/s/AKfycb.../exec',
  5. Save the file, commit, and push to GitHub: 
    git add cyrabelldental_v2.html
git commit -m "Set default GAS URL"
git push

After this, any device that opens the GitHub Pages URL will have the GAS URL pre-filled in Settings.

Step 3.3

Configure Clinic Settings

  1. Go to Settings → Clinic Info.
  2. Fill in: Clinic Name, Address, Phone, Email, Operating Hours, Logo.
  3. Go to Settings → Services → add your dental services with pricing (e.g., Cleaning ₱500, Extraction ₱800).
  4. Go to Settings → Reminders → configure which reminder types are active and their timing (e.g., 24 hours before appointment).
Step 3.4

Test Cloud Sync

  1. Go to Settings → Sync.
  2. Confirm the GAS URL is filled in. If not, paste your Apps Script URL and click Save.
  3. Click 🔄 Test Connection. You should see a green ✅ "Connected" message.
  4. Add a test patient (Patients → Add Patient).
  5. Click ⬆️ Push to Cloud (in the Sync card).
  6. Open your Google Sheet → check the patients tab — your test patient should appear.
✅

If the patient appears in the Sheet, cloud sync is working correctly.

4

Local Server Setup

The local server lets you run Cyrabell entirely on your clinic PC or USB drive — no internet required after initial setup.

Step 4.1

Install Node.js

  1. Go to nodejs.org and download the LTS version (e.g., v20).
  2. Run the installer. Accept all defaults.
  3. Verify installation — open Command Prompt (Windows) or Terminal (Mac/Linux): 
    node --version
npm --version
    Both commands should print a version number (e.g., v20.11.0).
Step 4.2

Start the Local Server

  1. Open Command Prompt / Terminal.
  2. Navigate to the local-server folder: 
    cd C:\cyrabell\local-server
    (Adjust the path to where you saved the Cyrabell folder.)
  3. Install dependencies (first time only): 
    npm install
  4. Start the server: 
    npm start
    You should see: 
    ╔══════════════════════════════════════════════════════╗
║       Simplified Dental Clinic Management Systems — Local Server              ║
╚══════════════════════════════════════════════════════╝

  URL  : http://localhost:3001
  Data : C:\cyrabell\local-server\data
  Files: C:\cyrabell\local-server\uploads
  5. Leave this terminal window open while using the app.
💡

To use a different port: PORT=8080 npm start (Mac/Linux) or set the PORT environment variable before running on Windows.

Step 4.3

Switch the App to Local Mode

  1. Open cyrabelldental_v2.html directly in Chrome (double-click the file, or drag it into the browser).
  2. Go to Settings → Sync.
  3. Click the 🖥️ Local Server toggle button.
  4. The URL field will be pre-filled with http://localhost:3001. Click Save.
  5. Click 🔄 Test Connection → should show ✅ "Local server connected".
  6. Click ⬆️ Push to Local to sync all existing data to the local server.
Step 4.4

USB / Fully Offline Setup

To run Cyrabell from a USB drive with no internet at all:

  1. Copy the entire CYRABELL_DentalMS folder onto your USB drive.
  2. On the USB, create a file called START_CYRABELL.bat (Windows) with this content: 
    @echo off
cd /d "%~dp0local-server"
start "" cmd /k "npm start"
timeout /t 3
start "" "%~dp0cyrabelldental_v2.html"
  3. Double-clicking START_CYRABELL.bat will start the local server AND open the app in the browser automatically.
  4. All data is stored in local-server/data/ on the USB.
⚠️

Node.js must be installed on the clinic PC even for USB mode. It does not run from the USB itself.

Backup & Restore

Step 5.1

Create a Backup

  1. Open backup-tool.html in your browser.
  2. Step 1 — Connect: Enter your GAS URL and click Test Connection.
  3. Step 2 — Download: Click Download Backup ZIP. This downloads all patient records, appointments, payments, and uploaded photos into a single cyrabell-backup-YYYY-MM-DD.zip file.
  4. Store the ZIP file in a safe place (Google Drive, external hard drive, email to yourself).
💡

Schedule weekly backups — every Friday before closing. Keep at least 4 weeks of backups.

Step 5.2

Restore from a Backup

  1. Open backup-tool.html.
  2. Scroll to Step 4 — Restore.
  3. Drag and drop your backup ZIP file into the drop zone.
  4. Click Push to Server. All data from the backup will be written back to the server.
🔴

Restore overwrites all current data on the server. Only restore if you are certain — there is no undo.

Troubleshooting

❌ "Test Connection" fails

  • Double-check the GAS URL is correct (no extra spaces, ends with /exec).
  • In Apps Script → Deployments, verify Who has access is set to Anyone.
  • Try opening the GAS URL directly in the browser — you should see a JSON response.

❌ Push/Pull shows no data in Google Sheet

  • Verify all 8 Sheet tabs exist with exact names (lowercase, no spaces).
  • Check the Apps Script Executions log for errors: Apps Script → Executions.
  • Make sure you created a New Deployment after any code changes.

❌ AI Assistant says "No API key"

  • Go to Settings → Sync → Gemini API Key card.
  • Re-enter the key and click ☁️ Save to Server.
  • If on local mode, the key is stored in local-server/data/_config.json.

❌ Local server — "Cannot connect to localhost:3001"

  • Make sure the terminal running npm start is still open and shows no errors.
  • Try http://127.0.0.1:3001 instead of localhost:3001 in the Sync URL field.
  • Check if another program is using port 3001: run netstat -an | find "3001" (Windows).
  • Start on a different port: PORT=8080 npm start and update the Sync URL to match.

❌ Photos not loading after switching modes

  • Cloud photos are stored in Google Drive — they only work in Cloud mode.
  • Local photos are stored in uploads/ — they only work in Local mode.
  • Use the backup tool to migrate photos between modes.

❌ "Unverified" login / can't log in

  • Default credentials are admin / admin123. Try these first.
  • If locked out, open browser DevTools (F12) → Console → type:
    localStorage.removeItem('cyrabell_users'); location.reload();
    This resets users to default. Warning: this removes all custom user accounts.

Frequently Asked Questions

Can multiple staff use the app at the same time?

Yes, if they are all using the cloud (GitHub Pages) URL. Each staff member opens the URL in their own browser. Data is synced through Google Sheets. In local mode, only one device can use the server at a time (unless you configure network access).

Does the app work on mobile phones / tablets?

Yes. The app is mobile-responsive. Use the GitHub Pages URL in Safari (iPhone) or Chrome (Android). Kiosk Mode is specifically designed for tablets.

How do I update the app when a new version is released?

Download the new cyrabelldental_v2.html, replace the old file in your repository, and push to GitHub. Re-paste your APP_DEFAULTS.GAS_URL if you set it. Your data in Google Sheets is not affected.

Can I change the clinic name shown inside the app?

Yes — go to Settings → Clinic Info → update the Clinic Name field. On Clinic Pro, you can also upload a custom logo.

What happens if I exceed the 500-patient limit on Starter?

The app will show a warning and block adding new patients. Existing data is not affected. Upgrade to Professional to remove the limit.

How do I set up Viber or Messenger reminders?

Viber and Messenger channels require Professional or Clinic Pro plan. Once unlocked, go to Settings → Reminders → enable the channels and enter the required credentials (Viber Business API token, Facebook Page Access Token).
